# Auth service — tests

- **`test_auth_shared.py`** — unit tests for the shared `auth` module (no DB):
  bcrypt hashing/verification (incl. malformed hash), JWT round-trip /
  expired / invalid-signature / garbage, `authenticate` header parsing, and the
  full **RBAC matrix** (`role_can`, `require_permission`, `require_role`).
- **`test_handler.py`** — the `function.handler` with the users repository
  monkeypatched (token/hash logic runs for real): register (default Viewer,
  validation, duplicate, elevated-role 403 without/with Admin, unknown role),
  login (success, wrong password, unknown user → 401), me (success, 401, 404),
  and routing errors (404/405/400/500).
- **`test_integration.py`** — real-PostgreSQL end-to-end: register → login → me,
  login by email, duplicate → 400, wrong password → 401, and role-assignment
  RBAC (elevated role needs an Admin token; a seeded Admin can create a Manager).
  Uses `.invalid` identifiers and cleans up; auto-skips when the DB is absent.

## Run

```sh
pip install -r backend/auth/requirements-dev.txt

# Unit tests only (no database needed)
pytest backend/auth/tests/test_auth_shared.py backend/auth/tests/test_handler.py

# Everything, including integration against local Postgres
IS_LOCAL=true pytest backend/auth/tests/
```

## Endpoints

| Method | Endpoint         | Auth        | Description                                  |
| ------ | ---------------- | ----------- | -------------------------------------------- |
| POST   | `/auth/register` | none/Admin* | Create a user (default role Viewer)          |
| POST   | `/auth/login`    | none        | Verify credentials, return a JWT             |
| GET    | `/auth/me`       | Bearer      | Return the current user from the token       |

\* Assigning any role other than Viewer requires an Admin bearer token.

## Applying RBAC in other services

The shared module syncs into every service. A CRUD handler enforces access with:

```python
import auth

try:
    principal = auth.authenticate(event)     # 401 if missing/invalid/expired
    auth.require_permission(principal, "delete")  # 403 if role can't delete
except auth.AuthError as err:
    return err.response
```

> `auth.py` and `postgres_service.py` here are generated by `bin/sync-shared.sh`
> from `backend/_shared/`. Run the sync (or a deploy) before packaging.
