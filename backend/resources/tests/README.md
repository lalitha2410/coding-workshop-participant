# Resources service — tests

Same structure as the projects and deliverables service tests:

- **`test_validation.py`** — unit tests for payload validation (pure, no DB):
  required `name`/`email`, email format, partial-update rules.
- **`test_handler.py`** — unit tests for the `function.handler` router with the
  repository monkeypatched. Covers routing, status codes
  (200/201/204/400/404/405/500), the **duplicate-email → 400** path, search
  pass-through, request parsing (API Gateway v1 + v2), and JSON serialization.
- **`test_integration.py`** — round-trips create → read → list/search → update →
  delete against a **real local PostgreSQL**, plus the UNIQUE-email behavior
  (duplicate create and update-to-existing-email both raise). Uses `.invalid`
  test emails and cleans up; auto-skips when the DB is unavailable.

## Run

```sh
python3 -m venv .venv && source .venv/bin/activate
pip install -r backend/resources/requirements-dev.txt

# Unit tests only (no database needed)
pytest backend/resources/tests/test_validation.py backend/resources/tests/test_handler.py

# Everything, including integration against local Postgres
IS_LOCAL=true pytest backend/resources/tests/
```

> `postgres_service.py` in this folder is generated by `bin/sync-shared.sh` from
> `backend/_shared/`. Run the sync (or a deploy) before packaging.
